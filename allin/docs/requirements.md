## 1. 项目概述

### 1.1 项目名称
**指数加仓提醒程序**（Index Drawdown Reminder）

### 1.2 项目背景
投资者需要定期跟踪指数（如沪深300、纳斯达克100等）的回撤幅度，以便在市场下跌到一定程度时执行加仓操作。传统方式需要手动查询指数走势图、计算回撤率，效率低且容易遗漏时机。本程序旨在自动化完成上述监控与提醒功能。

### 1.3 目标用户
个人指数基金投资者、定投策略使用者。

### 1.4 核心目标
在交易日 **14:30** 自动获取指定指数的实时净值，计算其与近 **6 个月**（约 180 天）内最高净值之间的回撤百分比，当回撤 ≥ 用户设定的阈值时，输出加仓提醒。

## 2. 功能需求

### 2.1 指数管理
- **支持多指数**：用户可配置一个或多个需要监控的指数（如沪深300、纳斯达克100、标普500、创业板指等）。
- **指数标识**：每个指数需有唯一代码或符号（如 `000300`、`NDX`），以适应不同数据源。

### 2.2 数据获取

#### 2.2.1 历史最高净值（基准点）
| 项目 | 说明 |
|------|------|
| **时间范围** | 从当前日期向前推 6 个月（约 180 天），包含起始日与结束日 |
| **数据粒度** | 日频数据，使用**每日最高价**（即当日盘中最高净值）作为判断依据。若数据源无法提供最高价，可退而使用收盘价 |
| **最高点定义** | 在上述时间窗口内，取所有交易日最高价的最大值 H_max |
| **数据更新周期** | 每日计算时重新拉取历史数据，确保"最近 6 个月"是滑动窗口 |

#### 2.2.2 当前实时净值
| 项目 | 说明 |
|------|------|
| **获取时刻** | 每个交易日 下午 14:30（以北京时间/本地时间为准） |
| **数据粒度** | 实时或近乎实时（延迟 ≤ 1 分钟）的指数点位 |
| **容错处理** | 若 14:30 数据暂不可用，则使用最近一次可用的日内数据或当日已有数据；若完全无法获取，则跳过当日提醒并记录日志 |

### 2.3 回撤计算

**公式**：
```
回撤率 = (H_max - P_current) / H_max × 100%
```
- `H_max`：近 6 个月最高净值
- `P_current`：当前实时净值
- 保留**两位小数**输出

### 2.4 加仓判断与提醒

- **触发阈值**：用户可自定义（默认 **10%**）。支持按指数分别设置不同阈值。
- **判断逻辑**：
  - 若 `回撤率 >= 阈值` → **建议加仓**
  - 否则 → **观望**
- **提醒方式**（至少包含以下一种）：
  1. 控制台输出（简单版本）
  2. 系统通知（Windows 通知 / macOS 通知 / Linux notify-send）
  3. *可选扩展：邮件、微信机器人推送*

### 2.5 定时执行
- 程序应支持自动在每日 **14:30** 运行检查。
- **运行频率**：每个交易日执行一次。
- 非交易日（周末、法定假日）自动跳过，可通过交易日历判断（推荐使用第三方库或接口）。

### 2.6 输出示例

#### 控制台/日志输出

```
[2026-06-09 14:30:01] 开始检查加仓条件
-----------------------------------------
指数：沪深300 (000300)
  近6个月最高点：5030.25 (日期: 2026-03-15)
  当前净值：4728.10
  回撤幅度：-6.00%
  阈值：10%
  结论：观望（暂不加仓）
-----------------------------------------
指数：纳斯达克100 (NDX)
  近6个月最高点：30762.50 (日期: 2026-04-02)
  当前净值：29414.00
  回撤幅度：-4.38%
  阈值：10%
  结论：观望
-----------------------------------------
检查完成。
```

#### 触发加仓时

```
[重要] 沪深300 已达到加仓条件！
  回撤幅度：12.50% (阈值 10%)
  建议执行加仓操作。
```

## 3. 非功能需求

### 3.1 数据源要求
| 要求 | 说明 |
|------|------|
| **免费可用** | 优先使用无需付费 API Key 的数据源（如 akshare、yfinance、baostock） |
| **覆盖范围** | 能同时提供 A 股指数（沪深300、中证500等）和主要美股指数（纳斯达克100、标普500） |
| **历史数据** | 至少能回溯 6 个月以上的日线数据（含最高价） |
| **实时数据** | 提供日内延迟报价（延迟 ≤ 1 分钟可接受） |

### 3.2 可靠性
- 网络异常时应有重试机制（重试 **3 次**，间隔 **5 秒**）
- 数据源接口变更时，程序应便于替换或修正
- 定时任务不应因某一次失败而停止后续任务

### 3.3 性能
- 单次检查（含数据拉取与计算）应在 **10 秒**内完成
- 内存占用 ≤ **200 MB**

### 3.4 易用性
- 提供配置文件（JSON 或 YAML）用于设置：
  - 监控的指数列表及代码
  - 各指数的加仓阈值
  - 提醒开关（是否控制台输出 / 系统通知）
- 提供简单启动脚本，用户无需修改代码即可配置

### 3.5 平台
- 优先支持 **Windows、macOS、Linux**（Python 跨平台）
- 提供对应平台的定时任务设置说明：
  - Windows：任务计划程序
  - macOS：launchd
  - Linux：crontab

## 4. 数据接口建议

| 指数类型 | 推荐数据源 | 历史数据方法 | 实时数据方法 |
|---------|-----------|-------------|-------------|
| A股指数（沪深300等） | `akshare` | `ak.stock_zh_index_daily(symbol="sh000300")` | `ak.stock_zh_index_spot()` |
| 港股指数（恒生） | `akshare` | `ak.stock_hk_index_daily()` | `ak.stock_hk_spot()` |
| 美股指数（纳斯达克100、标普500） | `yfinance` | `yf.download("^IXIC", period="6mo")`（含最高价） | `yf.Ticker("^NDX").info["regularMarketPrice"]` |

> **注意**：美股指数代码需确认，纳斯达克100常见代码 `^NDX`、`QQQ`。若需简化，可只支持 akshare 国内指数。

## 5. 边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| **非交易日 14:30** | 跳过检查，不执行任何操作 |
| **历史数据不足 6 个月**（如新上市指数） | 以实际可用数据天数计算最高点，并在日志中警告 |
| **14:30 实时数据获取失败** | 重试 3 次，仍失败则记录日志，使用前一日收盘价作为当前净值（并标注） |
| **指数代码无效** | 配置校验时提示错误，跳过该指数 |
| **网络断开** | 捕获异常，记录日志，退出本次检查（避免错误提醒） |
